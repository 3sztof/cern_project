NAME

       psSetProperties - Set the filter options of the FMC Process Monitor
                         Server

SYNOPSIS

       psSetProperties [-v] [-N DIM_DNS_NODE] [-m NODE_PATTERN...]
                       { --start | --stop } [-t | --tid TID]
                       [-u | --utgid UTGID_PATTERN] [-c | --cmd CMD_PATTERN]
                       [-C | --cmdline CMDLINE_PATTERN] [--a1 ARG1_PATTERN]
                       [--a2 ARG2_PATTERN] [--a3 ARG3_PATTERN]
                       [--a4 ARG4_PATTERN] [--a5 ARG5_PATTERN]
                       [--a6 ARG6_PATTERN] [--a7 ARG7_PATTERN]
                       [--a8 ARG8_PATTERN] [--a9 ARG9_PATTERN]

       psSetProperties { -V | --version } [-v] [-N DIM_DNS_NODE]
                       [-m NODE_PATTERN...]

       psSetProperties { -h | --help }

DESCRIPTION

       The command psSetProperties sets the filter properties to the FMC  Pro-
       cess Monitor Server psSrv(8), which are used to select the processes to
       be monitored by means of a dedicated DIM service.

       psSrv(8) keeps the filter properties in a filter  expression  which  is
       published  in  the  DIM service "/FMC/HOSTNAME/ps/monitored/properties"
       and can be read by means of did(1).

       The option --start adds a new (ORed) term  to  the  filter  expression,
       while  the option --stop removes an (ORed) term from the filter expres-
       sion.

       Different filter options specified in the same psSetProperties  --start
       command  are put in logical AND among them, while filter options speci-
       fied in different psSetProperties --start commands are put  in  logical
       OR among them.

       The DIM Name Server, looked up to seek for Task Manager Servers, can be
       chosen (in decreasing order of priority) by:

              1. Specifying the -N DIM_DNS_NODE command-line option;

              2. Specifying the DIM_DNS_NODE environment variable;

              3. Defining the DIM_DNS_NODE in the file "/etc/sysconfig/dim".

OPTIONS

       -h --help
              Get basic usage help from the command line.

       -v     Increase verbosity for debugging.

       -N DIM_DNS_NODE (string)
              Use DIM_DNS_NODE as DIM Name Server node.

       -m NODE_PATTERN (string, repeatable)
              Set the filter properties only to the process(es) running on the
              nodes  whose  hostname matches at least one of the wildcard pat-
              terns NODE_PATTERN (default: set the filter  properties  to  the
              process(es) running at all the nodes).

       -V --version
              For each selected node print the FMC Process Monitor Server ver-
              sion and the FMC version, than terminate.

       --start
              Add a new term to the filter expression used to select the  pro-
              cesses  to be monitored by means of a dedicated DIM SVC. Differ-
              ent options specified within the  same  psSetProperties  --start
              command  are put in logical AND among them, while options speci-
              fied in different psSetProperties --start commands  are  put  in
              logical  OR  among  them.  The  current filter expression can be
              viewed by means of the ".../monitored/properties" DIM SVC  (e.g.
              by using did(1)).

       --stop Remove  an  term  from  the filter expression used to select the
              processes to be monitored by means of a dedicated DIM  SVC.  The
              command  to  remove  a certain filter term must have exactly the
              same options used before to add the same filter term. E.g., if a
              filter term was added by means of the command:
              bash$ psSetProperties --start -c *Srv -u *Srv_u
              it can be removed by means of the command:
              bash$ psSetProperties --stop -c *Srv -u *Srv_u
              The  current  filter  expression  can  be viewed by means of the
              ".../monitored/properties" DIM SVC (e.g. by using did(1)).

       -t --tid TID (unsigned integer)
              Select/deselect the process which has TID as Thread  Identifier.

       -u --utgid UTGID_PATTERN (string)
              Select/deselect the process(es) whose UTGID matches the wildcard
              pattern UTGID_PATTERN.

       -c --cmd CMD_PATTERN (string)
              Select/deselect the process(es) whose  command  name  (basename,
              w/o path, of the executable image file, without arguments, even-
              tually truncated at 15 characters) matches the wildcard  pattern
              CMD_PATTERN.

       -C --cmdline CMDLINE_PATTERN (string)
              Select/deselect  the  process(es)  whose  command line (complete
              name of the executable image file, including path but  excluding
              arguments,  up  to PATH_MAX = 4096 characters) matches the wild-
              card pattern CMDLINE_PATTERN.

       --an ARGn_PATTERN (string)
              Select/deselect the process(es) whose n-th argument on the  com-
              mand-line matches the wildcard pattern ARGn_PATTERN (the digit n
              ranges in 1..9).

ENVIRONMENT

       DIM_DNS_NODE (string, mandatory if not defined otherwise, see above)
              Host name of the node which is running the DIM DNS.

       LD_LIBRARY_PATH (string, mandatory if not set using ldconfig)
              Must include the path to the libraries  "libdim"  and  "libFMCu-
              tils".

EXAMPLES

       Start monitoring the process whose command is "gnome-terminal"

       bash$ psSetProperties --start -c gnome-terminal

       Stop monitoring the process whose command is "gnome-terminal"

       bash$ psSetProperties --stop -c gnome-terminal

       Start monitoring the process whose UTGID is "ipmiSrv_u"

       bash$ psSetProperties --start -u ipmiSrv_u

       Stop monitoring the process whose UTGID is "ipmiSrv_u"

       bash$ psSetProperties --stop -u ipmiSrv_u

       Start  monitoring  the process whose UTGID matches the wildcard pattern
       "*Log" and whose command matches the wildcard  pattern  "*Srv"  on  the
       nodes whose hostname matches the wildcard pattern "lhcb*"

       bash$ psSetProperties --start -m lhcb\* -u \*Log -c \*Srv

       Stop  monitoring  the  process whose UTGID matches the wildcard pattern
       "*Log" and whose command matches the wildcard  pattern  "*Srv"  on  the
       nodes whose hostname matches the wildcard pattern "lhcb*"

       bash$ psSetProperties --stop -m lhcb\* -u \*Log -c \*Srv

       Start monitoring the process whose PID is 137

       bash$ psSetProperties --start -t 137

       Stop monitoring the process whose PID is 137

       bash$ psSetProperties --stop -t 137

       Start monitoring the process whose command line is
       "syslogd -m 0 -r -a /var/empty/sshd/dev/log":

       bash$ psSetProperties --start "--cmdline syslogd --a1 -m --a2 0 --a3 -r
       --a4 -a --a5 /var/empty/sshd/dev/log"

       Stop monitoring the process whose command line is
       "syslogd -m 0 -r -a /var/empty/sshd/dev/log":

       bash$ psSetProperties --stop "--cmdline syslogd --a1 -m --a2 0 --a3 -r
       --a4 -a --a5 /var/empty/sshd/dev/log"
