NAME

       cmGet - Read a file from one or more farm nodes through the FMC Config-
               uration Manager

SYNOPSIS

       cmGet [-v...]  [-N DIM_DNS_NODE] [-m NODE_PATTERN...]  [-q]
             [-o OUT_DIR] [-t RPC_TIME_OUT] -p FILE_PATH

       cmGet -V [-N DIM_DNS_NODE] [-m NODE_PATTERN]

       cmGet -h

INTRODUCTION

       The  FMC Configuration Manager is a tool to read, write and remove con-
       figuration files (more generally, ASCII files) to/from the farm  nodes.

       The  Process  Controller  Server, cmSrv(8), typically runs on each farm
       node and allows to access the files on the node it is running on.

       The Process Controller Clients cmSet(1), cmUnset(1)  and  cmGet(1)  can
       contact  one or more Process Controller Servers running on remote nodes
       to write files, remove files and read files respectively.

       File access can be restricted in 2 different ways  simultaneously:  (a)
       by  means  of  a username/hostname access control and (b) by means of a
       filename access control. See cmSrv(8) for more information.

DESCRIPTION

       The command cmGet(1) read the files at the absolute path FILE_PATH from
       all  the farm nodes whose hostname matches at least one of the wildcard
       patterns NODE_PATTERN and print their content either  to  the  standard
       output  stream (terminal), if the option -o OUT_DIR is omitted, or to a
       file in the directory OUT_DIR, if the option -o OUT_DIR is specified.

       Together with the file content, the command cmGet(1)  always  print  to
       the standard output stream also: (a) the date and time of the last mod-
       ification of the file; (b) the size of the file  (in  Bytes);  (c)  the
       file  permission  (in  octal notation); (d) the UID (User Identifier of
       the file owner); (e) the GID (Group Identifier of the file owner).

       The DIM Name Server, looked  up  to  seek  for  Configuration  Managers
       Servers, can be chosen (in decreasing order of priority) by:

              1. Specifying the -N DIM_DNS_NODE command-line option;

              2. Specifying the DIM_DNS_NODE environment variable;

              3. Defining the DIM_DNS_NODE in the file "/etc/sysconfig/dim".

OPTIONS

       -h     Get basic usage help from the command line, then terminate.

       -v     Increase verbosity for debugging.

       -N DIM_DNS_NODE (string)
              Use DIM_DNS_NODE as DIM Name Server node.

       -m NODE_PATTERN (string, repeatable)
              Operate  only with the nodes whose hostname matches at least one
              of the wildcard patterns NODE_PATTERN (default: operate with all
              the nodes).

       -V     For  each  selected  node  print  the  FMC Configuration Manager
              Server version and the FMC version, than terminate.

       -p FILE_PATH (string, mandatory)
              Read the files at path FILE_PATH from the selected nodes.

       -o OUT_DIR (string)
              Write the content of the remote files  to  local  files  in  the
              directory OUT_DIR. Local file are named NODE.FILE_PATH where the
              slash characters in  FILE_PATH  are  substituted  by  underscore
              characters.

       -q     Quiet  mode.  Do not print the file content. Print only the date
              and time of the last modification of the file, the size  of  the
              file  (in  Bytes),  the file permission (in octal notation), the
              UID (User Identifier of the file  owner),  and  the  GID  (Group
              Identifier of the file owner) and possible error messages.

       -t RPC_TIME_OUT (integer)
              Set  the timeout for the Remote Procedure Call to RPC_TIME_OUT .

ENVIRONMENT

       DIM_DNS_NODE (string, mandatory if not defined otherwise, see above)
              Host name of the node which is running the DIM DNS.

       LD_LIBRARY_PATH (string, mandatory if not set using ldconfig)
              Must include the path to the libraries  "libdim"  and  "libFMCu-
              tils".

EXAMPLES

       Read the remote files "/tmp/test.conf" at all the farm nodes registered
       with the current DIM name server and print their contents to  the  cur-
       rent  terminal  (together  with the date and time of the last modifica-
       tion, the size, the permission (in octal notation) and the UID and  the
       GID of the owner):

       bash$ cmGet -p /tmp/test.conf

       Read the remote files "/tmp/test.conf" at all the farm nodes registered
       with the current DIM name server and print to the current terminal only
       the  date  and  time of the last modification, the size, the permission
       (in octal notation) and the UID and the GID of the owner:

       bash$ cmGet -q -p /tmp/test.conf

       Read the remote files "/tmp/test.conf" at all the farm nodes registered
       with  the current DIM name server and print their contents to the local
       files hostname.tmp_test.conf in the current  directory;  the  date  and
       time of the last modification, the size, the permission (in octal nota-
       tion) and the UID and the GID of the owner are  still  printed  to  the
       current terminal:

       bash$ cmGet -p /tmp/test.conf -o .

       Read  the  remote  file "/tmp/test.conf" only at the farm node farm0101
       and print its content to the  current  terminal  (together  with  date,
       time, size, etc.):

       bash$ cmGet -m farm0101 -p /tmp/test.conf

       Use a wildcard pattern to select the nodes:

       bash$ cmGet -m "farm0[0-5]*" -m "farm07[135][0-5]" -p /tmp/test.conf
       bash$ cmGet -m "farm07[135][0-5]" -o . -p /tmp/test.conf
       bash$ cmGet -m "farm07[135][0-5]" -o /tmp -p /tmp/test.conf

       Check the version of the Configuration Manager Servers installed on the
       farm nodes:

       bash$ cmGet -V
