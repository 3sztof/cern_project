NAME

       tmKill - Send a signal to one or more running process(es) using the FMC
                Task Manager

SYNOPSIS

       tmKill [-v] [-N DIM_DNS_NODE] [-m NODE_PATTERN...]
              [--no-authentication] [-s SIGNAL] UTGID_PATTERN

       tmKill { -h | --help }

DESCRIPTION

       The  command  tmKill  sends  the  signal  SIGNAL  (default:  signal 15,
       SIGTERM, if the -s flag is omitted) to all the  processes  whose  UTGID
       matches the wildcard pattern UTGID_PATTERN at all the nodes whose host-
       name matches at least one of the wildcard patterns NODE_PATTERN.

       The DIM Name Server, looked up to seek for Task Manager Servers, can be
       chosen (in decreasing order of priority) by:

              1. Specifying the -N DIM_DNS_NODE command-line option;

              2. Specifying the DIM_DNS_NODE environment variable;

              3. Defining the DIM_DNS_NODE in the file "/etc/sysconfig/dim".

OPTIONS

       -h --help
              Get basic usage help from the command line.

       -v     Increase verbosity for debugging.

       -N DIM_DNS_NODE (string)
              Use DIM_DNS_NODE as DIM Name Server node.

       -m NODE_PATTERN (string, repeatable)
              Send  the  signal only to process(es) running on the nodes whose
              hostname matches at least one of the wildcard patterns NODE_PAT-
              TERN (default: send the signal to process(es) running at all the
              nodes).

       --no-authentication
              Do not send authentication string (compatible with  tmSrv,  ver-
              sion  <  2.0).  Works also with tmSrv version >= 2.0 if tmSrv is
              started with the --no-authentication command-line switch.

       -s SIGNAL (integer)
              Send the signal SIGNAL to the process(es) (default:  signal  15,
              SIGTERM).

ENVIRONMENT

       DIM_DNS_NODE (string, mandatory if not defined otherwise, see above)
              Host name of the node which is running the DIM DNS.

       LD_LIBRARY_PATH (string, mandatory if not set using ldconfig)
              Must  include  the  path to the libraries "libdim" and "libFMCu-
              tils".

EXAMPLES

       Send the signal SIGTERM to the processes with UTGID  "counter_0",  exe-
       cuting at all the nodes running the Task Manager Server registered with
       the current DIM DNS:

       bash$ tmKill counter_0

       Send the signal SIGTERM  to  the  processes  whose  UTGID  starts  with
       "count",  executing  at  all  the nodes running the Task Manager Server
       registered with the current DIM DNS:

       bash$ tmKill "count*"
       bash$ tmKill count\*

       Use another wildcard pattern to select the UTGID:.

       bash$ tmKill "count*[2-5]"

       Send the signal SIGTERM only to the processes with  UTGID  "counter_0",
       executing at nodes "farm0101" and "farm0102":

       bash$ tmKill -m farm0101 -m farm0102 counter_0

       Use wildcard patterns to select the UTGID and the nodes:

       bash$ tmKill -m "farm0*" -m "farm1*" "count*"
       bash$ tmKill -m farm0\* -m farm1\* count\*
       bash$ tmKill -m "farm0[3-7]01" "count*[2-5]"
       bash$ tmKill -m "farm0[3-7]??" "count*[2-5]"
       bash$ tmKill -m "farm010[1357]" counter_\*

       Send the signal 2 (SIGINT) instead of default signal 15 (SIGTERM):

       bash$ tmKill -s 2 counter_0
       bash$ tmKill -s 2 "count*[2-5]"
       bash$ tmKill -m farm0103 -m farm0201 -s 2 counter_0
       bash$ tmKill -m "farm0*" -s 2 "count*"

       Send  the  signal  to  all  the  processes  started by the Task Manager
       Servers registered with the current DIM DNS:

       bash$ tmKill "*"
       bash$ tmKill \*
       bash$ tmKill -s 2 "*"
       bash$ tmKill -s 2 \*
