NAME

       tmLs - List the process(es) started by FMC Task Manager (short list)

SYNOPSIS

       tmLs [-v | --debug] [-N | --dim-dns DIM_DNS_NODE]
            [-m | --hostname NODE_PATTERN...]  [-u | --utgid UTGID_PATTERN...]
            [-c | --compact] [-H | --header]

       tmLs { -V | --version } [-v | --debug] [-N | --dim-dns DIM_DNS_NODE]
            [-m | --hostname NODE_PATTERN...]

       tmLs { -h | --help }

DESCRIPTION

       The  command tmLs lists the processes, started by the FMC Task Manager,
       whose  UTGID  matches  at  least   one   of   the   wildcard   patterns
       UTGID_PATTERN, running on all the nodes whose hostname matches at least
       one of the wildcard patterns NODE_PATTERN.

       The list includes only the UTGIDs  of  the  processes  (i.e.  The  User
       assigned  unique  Thread  Group Identifier, set by the FMC Task Manager
       Server and used as a tag to identify the process).

       To get a more comprehensive list (including COMMAND, PID, OOM_SCORE and
       STATUS) the command tmLl(1) can be used instead.

       The output can be made more compact by means of the -c option.

       A sample tmLs output in standard format is:

               Node patterns list:  "*".
               UTGID patterns list:  "*".
                 Node pattern: "*":
                   Node: "farm0704". 16 process(es) have UTGID:
                    # UTGID
                    0 fmcLog
                    1 coalSrv_u
                    2 cpuinfoSrv_u
                    3 cpustatSrv_u
                    4 irqSrv_u
                    5 memSrv_u
                    6 nifSrv_u
                    7 cmSrv_u
                    8 psSrv_u
                    9 tcpipSrv_u
                   10 smartSrv_u
                   11 fsSrv_u
                   12 osSrv_u
                   13 gaudiLog
                   14 sysLog
                   15 tmSrv_u

       A sample tmLs output in compact format with header (-cH option) is:

               HOSTNAME N UTGID UTGID ...
               farm0307 16 fmcLog tmSrv_u cmSrv_u coalSrv_u cpuinfoSrv_u
                 cpustatSrv_u irqSrv_u memSrv_u nifSrv_u psSrv_u
                 tcpipSrv_u smartSrv_u fsSrv_u osSrv_u gaudiLog sysLog
               farm0912 17 fmcLog cmSrv_u pcSrv_u coalSrv_u cpuinfoSrv_u
                 cpustatSrv_u irqSrv_u memSrv_u nifSrv_u psSrv_u
                 tcpipSrv_u smartSrv_u fsSrv_u osSrv_u gaudiLog sysLog tmSrv_u
               farm0204 16 fmcLog coalSrv_u cpuinfoSrv_u cpustatSrv_u
                 irqSrv_u memSrv_u nifSrv_u cmSrv_u psSrv_u tcpipSrv_u
                 smartSrv_u fsSrv_u osSrv_u gaudiLog sysLog tmSrv_u

       If no UTGID_PATTERN is specified, it is set to "*".  If no NODE_PATTERN
       is specified, it is set to "*". The list printed by tmLs is updated not
       only periodically, but also whenever a Task  Manager  command  is  exe-
       cuted, and immediately after a process started by the current Task Man-
       ager istance has terminated for whatever reason.

       The DIM Name Server, looked up to seek for Task Manager Servers, can be
       chosen (in decreasing order of priority) by:

              1. Specifying the -N DIM_DNS_NODE command-line option;

              2. Specifying the DIM_DNS_NODE environment variable;

              3. Defining the DIM_DNS_NODE in the file "/etc/sysconfig/dim".

OPTIONS

   General Options
       -v --debug
              Increase verbosity for debugging.

       -N --dim-dns DIM_DNS_NODE (string)
              Use DIM_DNS_NODE as DIM Name Server node.

   Data Presentation Options
       -c --compact
              Print  the  process  list in a compact format (easier to parse):
              one host per line, no labels, no units, no separator lines.

       -H --header
              If the compact output format has been choosen, print  a  heading
              line with the output field labels.

   Data Selection Options
       -m --hostname NODE_PATTERN (string, repeatable)
              List  only  process(es)  running  on  the  nodes  whose hostname
              matches at least  one  of  the  wildcard  patterns  NODE_PATTERN
              (default: list process(es) running at all the nodes).

       -u --utgid UTGID_PATTERN (string, repeatable)
              List  only  the  process(es) whose UTGID matches at least one of
              the wildcard pattern UTGID_PATTERN (default: list all  the  pro-
              cesses  started  by  the  Task  Manager  Servers  running at the
              selected nodes).

   Accessory Options
       -h --help
              Get basic usage help from the command line, then terminate.

       -V --version
              For each selected node, print the FMC Task Manager  Server  ver-
              sion and the FMC package version, than terminate.

ENVIRONMENT

       DIM_DNS_NODE (string, mandatory if not defined otherwise, see above)
              Host name of the node which is running the DIM DNS.

       LD_LIBRARY_PATH (string, mandatory if not set using ldconfig)
              Must  include  the  path to the libraries "libdim" and "libFMCu-
              tils".

EXAMPLES

       List the processes having whatever UTGID, executing  at  whatever  node
       running the Task Manager Server registered with the current DIM DNS:

       bash$ tmLs
       bash$ tmLs -m "*" -u "*"
       bash$ tmLs -m \* -u \*

       Produce a more compact output:

       bash$ tmLs -c

       Produce  a  more  compact output with a header line showing field mean-
       ings:

       bash$ tmLs -cH

       List only processes having the UTGID "counter_0" or "gaudiLog" (running
       at whatever node):

       bash$ tmLs -u counter_0 -u gaudiLog

       Use a wildcard pattern to select the UTGIDs:

       bash$ tmLs -u "count*" -u "gaudi*"
       bash$ tmLs -u count\* -u gaudi\*
       bash$ tmLs -u "count*[2-5]" -u "count*[8-9]"

       Limit  the  list  to  the  processes  running  on  nodes "farm0101" and
       "farm0102":

       bash$ tmLs -m farm0101 -m farm0102

       Use a wildcard pattern to select the nodes:

       bash$ tmLs -m "farm01*"
       bash$ tmLs -m farm01\*
       bash$ tmLs -m "farm010[3-7]" -m "farm020[3-7]"
       bash$ tmLs -m "farm01?[3-7]" -m "farm02?[3-7]"

       Use a wildcard pattern to select both the nodes and the UTGIDs:

       bash$ tmLs -m "farm01*" -m "farm02*" -u "count*" -u "gaudi*"
       bash$ tmLs -m farm01\* -m farm02\* -u count\* -u gaudi\*
       bash$ tmLs -m "farm01?[3-7]" -m "farm02?[3-7]" -u "count*[2-5]"
