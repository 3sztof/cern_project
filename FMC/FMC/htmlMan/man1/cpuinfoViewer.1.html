NAME

       cpuinfoViewer - Get the CPU information about the farm PCs from the FMC
                       CPU Information Servers

SYNOPSIS

       cpuinfoViewer [-v | --debug] [-N | --dim-dns DIM_DNS_NODE]
                     [-m | --hostname NODE_PATTERN...]  [-c | --compact]
                     [-H | --header]

       cpuinfoViewer { -V | --version } [-v | --debug]
                     [-N | --dim-dns DIM_DNS_NODE]
                     [-m | --hostname NODE_PATTERN...]

       cpuinfoViewer { -h | --help }

DESCRIPTION

       Get the information about the CPUs  of  the  farm  PCs  whose  hostname
       matches  the  wildcard  pattern  NODE_PATTERN,  specified  using the -m
       NODE_PATTERN command line option (default:  all  the  nodes  registered
       with the DIM name server DIM_DNS_NODE). Information includes:

       VENDOR ID (string)
              The  CPU  brand identifier string. E.g.: GenuineIntel, Authenti-
              cAMD, CyrixInstead, GenuineTMx86.

       FAMILY (integer)
              The CPU family identifier. Value 6 stands for Intel Pentium III,
              AMD  K7,  Cyrix M2, VIA C3; value 15 stands for Intel Pentium IV
              and AMD K8, etc.

       MODEL (integer)
              The CPU model identifier. E.g.: value 1 can stand  for  P4  0.18
              um,  value  2 can stand for P4 0.13 um, value 3 can stand for P4
              0.09 um, value 5 can stand for AMD K8 Opteron 0.13 um.

       MODEL NAME (string)
              The CPU model identifier string. E.g.: "Intel(R) Pentium(R)  III
              CPU  family  1133MHz",  "Intel(R)  Xeon(TM)  CPU  2.40GHz", "AMD
              Athlon (TM) 64 Processor 3700+".

       STEPPING (integer)
              The sub-version identifier (a.k.a. revision number)  of  a  CPU.
              The stepping number increase with manufactory improvement or bug
              fixes of the same CPU model.

       CLOCK (float)
              The clock frequency of the CPU, measured in MHz (megahertz).

       CACHE SIZE (integer)
              The CPU cache size, measured in KiB (kibibytes).

       HYPER-THREAD (boolean)
              The Hyper-threading capability of the physical CPU package.

       MULTI-CORE (boolean)
              The Multi-core capability of the physical CPU package.

       PHYSICAL ID (integer)
              The physical package ID of the logical CPU.

       SIBLINGS (integer)
              The number of logical processors  (including  both  threads  and
              cores) in the physical package currently in use by the Operating
              System.

       CORE ID (integer)
              The core ID of the logical CPU.

       CPU CORES (integer)
              The total number of cores in the physical package  currently  in
              use by the Operating System.

       BOGOMIPS (float)
              This  number  is  a  very  raw  measure of the CPU computational
              speed. Etymology has origin by the words bogus (something  which
              is  a  fake, not genuine) and MIPS. MIPS is an acronym for "Mil-
              lions of Instruction per Second" and is the number  of  millions
              of  machine code instruction which a CPU is able to execute in a
              second. Bogomips are measured at boot time by running a  program
              loop  which  does  absolutely  nothing  (hence the definition of
              Bogomips as "the number of million times per second a  processor
              can  do  absolutely  nothing"). Bogomips ranges from 0.02 (Intel
              8088 at 4.77 MHz) to 6717.44  (Intel  Pentium  4  at  3.4  GHz).
              Bogomips/CPU  clock ratio ranges from 0.004 (Intel 8088) to 3.97
              (Intel Xeon with hyper-threading).

       The DIM Name Server DIM_DNS_NODE, looked up to seek for FMC CPU  Infor-
       mation Servers, can be chosen (in decreasing order of priority) by:

              1. Specifying the -N DIM_DNS_NODE command-line option;

              2. Specifying the DIM_DNS_NODE environment variable;

              3. Defining the DIM_DNS_NODE in the file "/etc/sysconfig/dim".

OPTIONS

       -h --help
              Get basic usage help from the command line, then terminate.

       -v --debug
              Increase verbosity for debugging.

       -N --dim-dns DIM_DNS_NODE (string)
              Use DIM_DNS_NODE as DIM Name Server node.

       -m --hostname NODE_PATTERN (string, repeatable)
              Print  only the CPU information of the farm nodes whose hostname
              matches at least  one  of  the  wildcard  patterns  NODE_PATTERN
              (default:  print the CPU information of all the nodes registered
              with the DIM name server DIM_DNS_NODE).

       -V --version
              For each selected node print the FMC CPU Information Server ver-
              sion and the FMC version, than terminate.

       -c --compact
              Print  the  CPU information in compact format (easier to parse):
              one core per line, no labels, spaces inside fields  replaced  by
              underscores.

       -H --header
              If  the  compact output format has been choosen, print a heading
              line with the output field labels.

ENVIRONMENT

       DIM_DNS_NODE (string, mandatory if not defined otherwise, see above)
              Host name of the node which is running the DIM DNS.

       LD_LIBRARY_PATH (string, mandatory if not set using ldconfig)
              Must include the path to the libraries  "libdim"  and  "libFMCu-
              tils".

EXAMPLES

       Print the CPU information about all the farm nodes:

       bash$ cpuinfoViewer

       Print output in compact format with header:

       bash$ cpuinfoViewer -c -H
       bash$ cpuinfoViewer -cH
       bash$ cpuinfoViewer --comp --hea

       Print only the CPU information of selected nodes:

       bash$ cpuinfoViewer -m farm0101 -m farm0102
       bash$ cpuinfoViewer -host farm0101 -host farm0102

       Print only the CPU information of wildcard-selected nodes:

       bash$ cpuinfoViewer -m "farm01*" -m "farm02*"
       bash$ cpuinfoViewer -m "farm01?[1357]"
       bash$ cpuinfoViewer -m "farm01?[1-5]"
       bash$ cpuinfoViewer --host "farm01[0-5][1-36-9]"

       Print the Server version and the FMC package version of the running FMC
       CPU Information servers registered with the current DIM name server:

       bash$ cpuinfoViewer -V
